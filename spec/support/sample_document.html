Skoogle Docs

An un-arranged marriage between Google & Skookum

Why we made this

One of my first tasks here at Skookum was to create a template that could be used for internal documents and client deliverables. It needed to be clean, functional and easily created by individuals and/or groups.

Platform Decision

The obvious platform choices were Google Docs, Apple Pages and Microsoft Word.  All choices have strong selling points but also lack certain capabilities that were key to our needs. Below are some high level pros & cons to each platform (without any emotional ties):

Google Docs

Pros

  • Amazing collaboration capabilities
  • Real time comments and edits between team members
  • Available on multiple platforms
  • Easy to use
  • Version control
  • Save document in multiple formats (including HTML)

Cons

  • Doesn’t support custom fonts
  • Easy to “break” the brand.  User would need to create with care (a process that could be very time consuming)
  • Future updates to brand will require changes to each individual document
  • Large documents will create large files sizes
  • Updates will need to be re-sent to team/client

Apple Pages

Pros

  • Native Applications & Web Application
  • Custom Fonts
  • Easy to use

Cons

  • Poor collaboration capabilities
  • Native Apps not available on Windows or Chrome systems
  • Cannot export document as HTML
  • Easy to “break” the brand.  User would need to create with care (a process that could be very time consuming)
  • Future updates to brand will require changes to each individual document
  • Large documents will create large files sizes
  • Updates will need to be re-sent to team/client

Microsoft Word

Pros

  • None

Cons

  • Many

Which to choose

None of the above.  Well, not quite… Google wins, but not fully.  I kept coming back to the users need to “create with care” and the fact that documents would need to be re-worked in the future if our brand got tweaked.  Google’s collaborative feature is just so strong that I felt it needed to be part of the document creation process.  It keeps the entire team (large or small) synced at all times across all devices in real time, allows comments and eliminates the worry of lost content with version control.

Lightbulb Moment

One particular “pro” that was part of Google Docs was it’s ability to save a document to HTML… BAM!  

What if we could leverage all of Google’s collaborative features, eliminate the user’s need to make the document “pretty” while adding custom fonts and a single CSS file that could update the look and feel of every document we make in the future?  Add to the mix the ability to provide a link to a living responsive web document and we’ve combined Skookum ingenuity with Google power… Skoogle Docs :)

Going Digital

After receiving positive feedback, it was decided to expand on Skookum’s deliverables and take them to the next level with similar methods, providing a more coherent and professional end result. Being a digital company on the forefront of technology, it is our belief that producing deliverables in a completely digital format will act as an innovative, useful and impressive method of delivering work.

Although a lot of clients will be custom to handoff in .pdf or print ready formats, providing the additional digital experience can provide a lot of benefits for both Skookum internally and the client.

Client

Web Experience

Instead of downloading and opening a .pdf in a 3rd party program, clients can simply load the deliverable in browser.

Easier Navigation

Clients can find and navigate sections easier with the ability to search.

Multimedia Capabilities

Interactive elements can be included to provide a richer experience.

Multiple Devices

Ability to reformat appropriately on multiple devices for access anywhere

Exportable

Clients will still want “hard” copies.  Built into Skoogle Docs is the ability to print/save a PDF version of the document.  This version has Headers, Footers and page numbers.

Internal

Updateable

Each deliverable will be treated like a living document that can be updated and changed easily. This will reduce steps in the deliverable publishing process and allow employees to focus on other work.

Consistency

Restricting the design and automatically applying styles will help keep documents consistent across the company, allowing Skookum to maintain it’s brand.

Reduced Design Role

Backwards and forwards discussions between the content producers and designers will be significantly reduced, allowing content providers to make changes without the need for a designer.

Online Storage

All text, images and assets within the file are stored on Google Drive.  Only the base code is hosted by Skookum so we won’t incur any storage bloat issues.

Future Proof

One CSS file to rule them all. If we choose to make a change to our brand (massively or minimally) we can make the changes to our CSS (Scss) and deploy it.  Magically all of our new documents will be branded correctly.  But wait… there’s more.  All of our previous documents will be updated too!  Brand consistency across the old and the new.  Now that’s Kool-Aid!

Footnotes & Links

Users can link to other sections within the document.  Google docs also allows footnotes[1] that can be used for notes or links[2].  These footnote details exist on the bottom of the page noted in Google Docs.  In Skoogle Docs, they’ll exist at the bottom of the document before the footer.

Hacked Styles (The special sauce)

I’ve created an easy way to manipulate the appearance of List Items. This is accomplished by creating a Bulleted List and adding a “#[NAME]” as the last list item. When the Skoogle Doc is created, the #[NAME] is removed from the mix leaving only a polished appearance.

Below is a set of examples to show what is possible.  The options are limitless to what we create using this method.  That said, we’ll want to be selective and choose to create styles for things that will be reused for multiple scenarios.

Definitions

Odd numbered bullets (1,3,5…) are the definition titles.  Even numbered bullets (2,4,6…) are the corresponding definitions.
